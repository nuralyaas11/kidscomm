## Overview

**KidsComm** is a Multi-Page Application (MPA) built with modern static web technologies and powered by a cloud database to store student achievement records. It uses a **Serverless Architecture** — no PHP or local database servers (like XAMPP) required.

---

## ✨ Features

### 🎨 Kid-Friendly Interface
Cheerful, interactive, and fully responsive UI/UX built with Tailwind CSS.

### 👥 Role-Based Access System

| Role | Access |
|------|--------|
| **Student** (`customer`) | Play games, answer quizzes, view personal scores |
| **Teacher** (`admin`) | Access Admin Panel, view student list, monitor all scores |

### 🎮 Mini-Game Modules

- **Game 1 — Picture Matching**: Match situation images with the correct communication response or phrase.
- **Game 2 — Word Arrangement**: Rearrange scrambled word blocks into a complete, correct communication sentence.

### 📝 Quiz Module
Interactive objective quizzes with an automatic scoring system.

### 📊 Dashboard
Displays activity summaries and score records powered by Firebase Firestore.

## 🗄️ Database Schema

This project uses a **NoSQL** structure in **Firebase Cloud Firestore** (Collection → Document → Fields).

### Collection: `users`

Stores user profiles and roles for login authentication.

| Field | Type | Description |
|-------|------|-------------|
| `email` | `string` | User email (e.g. `"admin@kidscomm.com"`) |
| `nickname` | `string` | Student display name (e.g. `"Ahmad"`) |
| `role` | `string` | User role: `"admin"` or `"customer"` |
| `createdAt` | `timestamp` | Account registration date and time |

> **Document ID**: `uid` — automatically retrieved from Firebase Authentication.

---

### Collection: `scores`

Stores all game and quiz records for display on the Student Dashboard and Admin Panel.

| Field | Type | Description |
|-------|------|-------------|
| `userId` | `string` | References the `uid` of the player |
| `nickname` | `string` | Stored alongside for easy admin display without extra queries |
| `activityType` | `string` | Activity type: `"game1"`, `"game2"`, or `"quiz"` |
| `score` | `number` | Score obtained (e.g. `100`) |
| `playedAt` | `timestamp` | Date and time the activity was completed |

> **Document ID**: Auto-ID — automatically generated by Firestore.

---

### Collection: `content` *(Optional)*

Allows admins to update quiz questions without modifying HTML code.

| Field | Type | Description |
|-------|------|-------------|
| `questions` | `array of objects` | Contains `questionText`, `options` (array), and `correctAnswer` |

> **Document ID**: `quiz_data`

## ⚙️ Getting Started

### Step 1: Firebase Setup

1. Go to the [Firebase Console](https://console.firebase.google.com/) and create a new project.
2. Enable **Firebase Authentication** → select the **Email/Password** sign-in method.
3. Enable **Cloud Firestore** → start in **Test Mode** for development, then update security rules before going live.
4. Register a **Web App** in Firebase Console and copy the `firebaseConfig` object.

---

### Step 2: Configure the Project Locally

1. Clone this repository.
2. Open `firebase-config.js` in the root directory.
3. Replace the placeholder values with your Firebase API keys from Step 1.

```js
// firebase-config.js
const firebaseConfig = {
    apiKey: "YOUR_API_KEY",
    authDomain: "YOUR_PROJECT_ID.firebaseapp.com",
    projectId: "YOUR_PROJECT_ID",
    storageBucket: "YOUR_PROJECT_ID.appspot.com",
    messagingSenderId: "YOUR_SENDER_ID",
    appId: "YOUR_APP_ID"
};
```

---

### Step 3: Create the First Admin Account

> ⚠️ The system blocks users with no assigned role from logging in. You **must** manually create the first admin account.

1. In Firebase Console, go to **Authentication** → click **Add User**. Enter the admin email and password.
2. Copy the generated **User UID**.
3. Go to **Firestore Database** → create a new collection named `users`.
4. Add a new document — paste the User UID as the **Document ID**.
5. Add the following fields:

| Field | Type | Value |
|-------|------|-------|
| `email` | string | *(admin email)* |
| `nickname` | string | *(admin display name)* |
| `role` | string | `"admin"` |

---

### Step 4: Run Locally

No XAMPP or localhost PHP server needed. Simply:

1. Install the **Live Server** extension in VS Code.
2. Right-click `index.html` → **Open with Live Server**.
3. Log in using the admin account created in Step 3.

## 🚀 Deployment

Deploy for free using **GitHub Pages**:

1. Push all project files to your GitHub repository.
2. Go to the repository **Settings** → **Pages**.
3. Under *Build and deployment*, set:
   - **Source**: `Deploy from a branch`
   - **Branch**: `main`
   - **Folder**: `/ (root)`
4. Click **Save** and wait a few minutes.

Your KidsComm app will be publicly accessible via your GitHub Pages URL! 🎉
